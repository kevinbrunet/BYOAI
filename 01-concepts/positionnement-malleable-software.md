# Positionnement — le BYOIA, versant organisationnel du malleable software

**Décision (2026-07-11) : positionnement adopté pour le livre.**

## La formule

Le BYOIA est le **versant organisationnel du malleable software**. Le courant malleable software (Tchernavskij 2019, Litt 2023, Ink & Switch 2025) fournit la fondation intellectuelle : les LLM rendent enfin possible l'end-user programming, le logiciel doit redevenir de l'argile plutôt que de l'électroménager scellé. Le livre cite cette généalogie et occupe le terrain qu'elle laisse vide : l'entreprise.

## Le partage des terrains

| | Malleable software | BYOIA |
|---|---|---|
| Sujet | L'individu souverain sur ses outils | Le collaborateur dans une organisation |
| Question centrale | Comment remodeler son logiciel ? | Comment gouverner le remodelage sans recréer le Shadow IT ? |
| Partage | Communal creation (entre pairs) | [Cycle de promotion](./cycle-promotion-specifique.md) (avec transfert de maintenance) |
| Garanties | Absentes (hors sujet pour eux) | [Frontière fiduciaire](../05-questions-ouvertes/doctrine-frontiere-si-interface.md), traçabilité, [conformité](./conformite-par-design.md), CSE |
| Limite de croissance | Non traitée | [Fenêtre de complexité](./cycle-promotion-specifique.md) |

## Ce que ça apporte au livre

1. **Une généalogie citables** : Shirky (situated software, 2004), Sloan (home-cooked meal, 2020), Tchernavskij (2019), Litt (2023), Ink & Switch (2025). Le livre ne part pas de zéro — il prolonge un courant identifié, ce qui le crédibilise.
2. **La continuité métaphorique** : leur imaginaire est domestique (home kitchen, home-cooked meal) ; le [récit d'ouverture](./captivite-logicielle-happy-path-social.md) parle de recettes de cuisine à la machine à café. La métaphore file d'un bout à l'autre : la recette de cuisine officieuse (le contournement subi) devient la cuisine de maison outillée (le spécifique assumé) — puis le plat qui entre à la carte commune (la promotion).
3. **Une différenciation nette** : à la question "en quoi est-ce nouveau ?", la réponse est précise — le malleable software n'a ni frontière fiduciaire, ni cycle de promotion, ni fenêtre de complexité, ni réponse juridique. C'est exactement le contenu original du livre.

## Vigilance

- Ne pas se contenter de "Litt + gouvernance" : le livre doit démontrer que le versant organisationnel change la *nature* du problème (la confiance d'autrui, l'audit, le droit du travail), pas seulement son échelle.
- Vérifier ce que Litt/Notion publient d'ici la parution — le courant évolue vite et Notion est en position de produire le "malleable enterprise" avant le livre.
