# Objection — La frontière SI/interface tient-elle si l'IA génère du code des deux côtés ?

## Le problème

"Les règles de gestion à la charge du SI" — est-ce que ça reste vrai si l'IA génère aussi du code côté SI ? Le post pose une frontière nette (SI = données + règles / utilisateur = interface), mais l'IA peut potentiellement toucher aux deux couches. Un lecteur technique qui connaît le sujet en profondeur peut voir une tension.

## Statut : TRANCHÉ (2026-07-11, rédaction ch. 5)

Troisième voie retenue, plus forte que les deux envisagées : **le critère porte sur l'artefact, pas sur l'auteur**. Peu importe qui écrit le code (humain, IA, les deux) — c'est ce que l'artefact touche qui gouverne : s'il touche du fiduciaire (les 5 critères), régime complet ; sinon, régime libre. La frontière est indifférente à l'auteur du code. Voir [chapitre 5](../07-manuscrit/chapitre-05.md).

## Deux façons de trancher (historique)

1. L'assumer en une phrase : "la frontière est organisationnelle, pas technique — l'IA peut opérer des deux côtés, la gouvernance diffère."
2. La laisser implicite (risque : un lecteur technique la soulève en premier).
