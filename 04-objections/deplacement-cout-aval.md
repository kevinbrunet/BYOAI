# Objection — Déplacement du coût vers l'aval

## Le problème

"Le coût du code s'effondre" est un raccourci risqué. Le coût de *production* du code chute, oui. Le coût de validation, maintenance et responsabilité ne chute pas au même rythme — et dans un contexte régulé (ex. IEC 62304 ⚠ *à vérifier selon le secteur exact concerné*), c'est même l'inverse : plus d'outils spécifiques = plus de surface à qualifier.

Le post tel quel laisse entendre un gain net alors que c'est un déplacement du coût vers l'aval.

## Deux options pour rester technically defensible

1. Assumer explicitement le déplacement de coût (ce n'est pas un gain net, c'est un changement de qui paie et quand).
2. Couper l'argument s'il n'est pas possible de le nuancer correctement dans le format visé.
