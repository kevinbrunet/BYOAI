# Argument 1 — L'économie d'échelle sur le code s'effondre

## Le raisonnement ancien

L'argument reposait sur un principe simple : le coût fixe de développement doit être amorti sur le plus grand nombre d'utilisateurs possible. Mutualiser le développement divise le coût par le nombre d'utilisateurs.

## Ce que l'IA change

Si le coût fixe de production du code tend vers zéro, l'argument disparaît. Produire dix outils légèrement différents pour dix besoins légèrement différents n'est plus structurellement plus cher que d'en produire un seul.

## À nuancer

"Le coût du code s'effondre" est un raccourci risqué : le coût de *production* du code chute, mais le coût de validation, de maintenance et de responsabilité ne chute pas au même rythme. Dans un contexte régulé, c'est même l'inverse : plus d'outils spécifiques = plus de surface à qualifier. Voir [déplacement du coût vers l'aval](../04-objections/deplacement-cout-aval.md) — c'est un déplacement du coût vers l'aval, pas un gain net, sauf à l'assumer explicitement.
